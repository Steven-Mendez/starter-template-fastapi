When I surface a decision with options and a firm recommendation, the user wants me to **proceed with the recommended option without stopping to ask**.

**Why:** Stated directly 2026-05-16 ("Siempre lo recomendado") after I halted ETAPA I step 7 for a 3-way choice. The user runs autonomous goals and treats my recommendation as the default they trust; halting for confirmation on every judgment call breaks the autonomous flow they asked for.

**How to apply:**
- Still *surface* the decision, the options, the recommendation, and the rationale (transparency matters, especially for destructive/irreversible or premise-contradicting actions).
- Then **act on the recommended option** and keep going — do not block waiting for a letter/confirmation.
- **Exception that still warrants a real stop:** genuinely destructive or irreversible actions with no safe default (e.g. deleting working code on a false premise *with no recommended safe path*). In step 7 there *was* a safe recommended path (drift-fix, no deletion) — so proceeding was correct. If the recommendation itself is the risky path, still pause.
- Always **write the chosen decision down durably** (memory + the OpenSpec change proposal + a ROADMAP note), not just in chat — the user explicitly wants decisions traceable. See [[step7-s3-false-premise]], [[roadmap-workflow]].
