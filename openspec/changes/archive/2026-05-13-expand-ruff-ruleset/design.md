## Context

Ruff's current `select = ["E","F","I","B"]` is conservative. Existing `# noqa: S, DTZ003, PLC0415, BLE001, ARG001/2` comments scattered through the source prove contributors expect tighter rules; the config just hasn't been updated.

## Decisions

- **Rule families adopted** (one PR):
  - `B` (flake8-bugbear) — already on; keep.
  - `S` (bandit) — security hygiene (assert-in-prod, hardcoded passwords, subprocess shell, unsafe yaml.load, etc.).
  - `ASYNC` (flake8-async) — catches `time.sleep` in async, missing-await, blocking IO in coroutines.
  - `PERF` (perflint) — list-comprehension and `try/except in a loop` perf antipatterns.
  - `RET` (flake8-return) — unnecessary `else` after `return`, inconsistent return paths.
  - `SIM` (flake8-simplify) — collapsible-if, `dict.get` over key-lookup-fallback, etc.
  - `UP` (pyupgrade) — auto-modernises syntax (e.g. `Optional[X]` → `X | None`).
  - `PL` (pylint subset Ruff supports) — common antipatterns; pair with selective ignores.
  - `C4` (comprehensions), `RUF`, `PT` (pytest), `TRY` (tryceratops), `ARG`, `DTZ` (datetimez) — already in the proposal scope; keep.
- **Per-rule ignores with rationale**:
  - `PLC0415` (import-outside-top-level) — already used; keep as a global ignore. Rationale: composition-root lazy imports avoid circular import cycles.
  - `S101` (assert) — ignored under `**/tests/**` and `**/conftest.py`. Rationale: pytest depends on `assert`.
  - `S105`, `S106` (possible-hardcoded-password) — ignored under tests. Rationale: test fixtures use literal credentials.
  - `ARG` — ignored under tests. Rationale: pytest fixtures pass through args by name; unused params are idiomatic.
  - `RUF012` (mutable-class-default) — ignored under `src/features/**/adapters/outbound/persistence/sqlmodel/models.py`. Rationale: SQLModel idiom uses `dict` / `list` class defaults.
  - `TRY003` (long messages outside exception class) — ignored project-wide. Rationale: domain errors are deliberately string-formatted at the raise site.
- **`alembic/versions/*.py`** — ignore all rules (autogenerated; ruff fights Alembic's templates).
- **Don't enable `D` (docstring) yet** — noisy, separate proposal if/when adopted.

## Non-goals

- Not adding per-file `# noqa` blanketly — every new `# noqa` must name the rule code and (where non-obvious) include a one-line reason.
- Not enabling the `D` (pydocstyle) family in this change.
- Not authoring per-file ignores for individual `src/` files; only the four globs in §1.3 of `tasks.md` are introduced.
- Not changing `target-version` or `line-length`.
- Not adopting `RUF100` cleanup as a separate gate beyond what `make lint` already enforces.

## Risks / Trade-offs

- One-time cleanup PR can be large. Mitigation: `make lint-fix` auto-fixes `UP`, `SIM`, `C4`, `RET` (70%+). The remainder is mostly `S` (security) which is the value.
- `PL` family can be noisy — start with the subset that auto-fixes; review the per-rule findings before locking the config.
